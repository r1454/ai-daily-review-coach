# 测试数据 — 样例日记录

以下是用于测试 AI每日复盘教练 的样例日记录输入。

## 样例 1：学习场景

```
今天上午去上了大数据课，讲了 Spark 的 RDD 转换操作，半懂不懂的。
下午在图书馆复习，试着写了一个简单的 WordCount，跑通了但不太理解原理。
晚上和室友打了两把游戏，然后看了一会儿 B 站就睡了。
```

## 样例 2：工作/项目场景

```
今天主要在公司写那个 API 接口，花了一上午调一个奇怪的 bug，最后发现是
数据库字段名拼错了（user_name 写成了 usre_name），气得不行。
下午和产品经理开了个需求评审会，她又临时加了两个需求，感觉 deadline 赶不上了。
晚上加班到 9 点，回家啥也不想干。
```

## 样例 3：混合场景

```
白天效率还行，上午把作业写完了。中午和同学出去吃了火锅，聊了聊考研的事，
感觉压力挺大的，大家都开始准备了，我还没想好要不要考。
下午看了一小时书就看不进去了，刷手机刷到晚上。
晚上突然想起明天有早课，赶紧设了 3 个闹钟。
```

## 样例 4：简短场景

```
今天没什么特别的事，上课、吃饭、睡觉。唯一特别的是晚上跑了 5 公里，
好久没跑了，跑完挺爽的。
```

## 样例 5：多事件场景

```
今天发生了好几件事：
1. 早上组会汇报上周进展，被导师说进度太慢，让我这周必须出结果
2. 下午帮师姐处理数据，学到了一个新的 Python 库（polars），感觉比 pandas 快很多
3. 晚上收到实习面试通知，下周二，紧张
4. 睡前妈妈打电话说外婆住院了，有点担心
```

## 预期 AI 追问示例（对应样例 1）

基于样例 1 的日记录，AI 应生成类似以下风格的追问：

1. （动机角度）你说上课"半懂不懂"，是你觉得 Spark 本身难，还是老师讲的方式让你跟不上？有没有试着课后问老师或同学？

2. （行为/决策角度）WordCount 虽然跑通了但不理解原理——你觉得是 RDD 的概念没搞懂，还是代码里某个具体步骤卡住了？如果再花 20 分钟去看文档，你觉得能解决吗？

3. （结果/影响角度）晚上选择打游戏看 B 站而不是继续学 Spark——如果明天上课又讲新内容，你今天没搞懂的会不会滚雪球？你觉得什么方式能帮自己打破这个循环？

---

*这些样例数据用于验证 AI 追问的质量和针对性。好的追问应该基于具体内容深挖，而不是问泛泛的问题。*
